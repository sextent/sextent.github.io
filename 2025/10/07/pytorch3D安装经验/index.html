pytorch3D安装经验
更新于 2025-11-04 教程

我在安装程序运行环境时，遇到了pytorch3d在windows系统上的编译不成功的问题，这耗费了我国庆宝贵时光中的一整天。我非常生气，所以我决定写一篇博客来记录安装过程。这篇博客将介绍如何安装pytorch3D，以及我在安装过程中遇到的问题和解决方案。

前置环境说明

我使用cu121+pytorch2.1.0+torchvision0.16.0作为基本开发环境

安装过程

  1. 我首先尝试使用pip安装pytorch3d，但是安装过程中出现了编译错误。我尝试了多次，但是都没有成功。于是，我决定使用conda来安装pytorch3d。

  2. 我使用conda安装pytorch3d，但是安装过程中发现conda好像没有提供对应环境的安装包，一直会报错

  3. 到了这一步，该克隆源码安装了。

按照项目的安装教程 ，克隆了源代码仓库然后进入源码目录，运行setup.py文件。
但是，在编译过程中，仍然遇到了和直接使用pip一样的错误。

之前配环境的时候已经遇到过这个问题了。当时以为是ninja的问题，实际上是cuda的位置未指定导致的。ninja实际上是一个构建系统，可以把它理解为一个更现代、更快速的 make 工具。PyTorch 的 C++ 扩展在编译时，如果检测到你环境里安装了ninja，会优先使用它来进行并行编译，因为速度更快。而出问题的函数是PyTorch的构建脚本（cpp_extension.py），所以是调用ninja进行编译时出了问题。根据之前的经验，原理大概是pytorch3d的编译同时深度依赖NVIDIA的CUDA工具链。如果CUDA的路径没有被正确地传递给构建系统，那么在生成编译指令（build.ninja文件）或者执行这些指令时，完全可能出现各种奇怪的底层错误。
而我们之前的排查都集中在C++编译器（MSVC）和构建系统（Ninja）上。所以我按照之前的做法先确认终端中的cuda路径是否正确。

新报错为

这次问题很简单。

  • 第一行告诉我们： 你的编译系统正在使用的C++编译器是 Visual Studio 14.0，也就是 Visual Studio 2015。
  • 第二行告诉我们： PyTorch的源代码明确要求，必须使用支持 C++17 标准的编译器。

但是我安装的是VS2022的发行版，不应该回到2015的问题。所以我又想到之前为了编译包安装了MCVS的老版本，是不是直接用它来编译了？那么就需要验证where cl.exe了。我先运行了vs2022的终端，我想这里面应该不会使用老版本的工具了，但是运行同样的安装代码时报错：

所以还是找不到编译器。清除了旧版本，重新安装新版本之后再运行，

来看看AI怎么说：

NVIDIA的CUDA编译器(nvcc.exe) 在尝试编译代码时，明确地告诉我们：
“我只认识 2017 到 2019 版本的 Visual Studio。你现在系统里配置的这个版本（VS 2022），我根本不认识，也无法使用它来编译代码！”
nvcc.exe 是编译CUDA代码（.cu 文件）的主帅，但它需要一个C++编译器（cl.exe）来当它的副将，处理代码中标准的C++部分。你的 CUDA Toolkit 12.1 版本在设计时，它的“副将名单”里只写了 VS2017 和 VS2019，没有 VS2022。
所以，整个编译流程在 nvcc 这一步就直接罢工了。

AI让我装2019的VS。但我还想自己试试，首先在VS终端中输入cl.exe是有输出的，但在powershell里面就不行。
关注powershell里的报错：

我们的C++ 编译器版本太新了，超出了当前 CUDA 12.1 工具包所能识别和兼容的范围。所以，虽然错误信息说支持到 2022，但它指的是 CUDA 12.1 发布时 的 VS 2022 版本。你的 Visual Studio Build Tools 自动安装了最新的版本，这个版本对于 CUDA 12.1 来说过于前卫了。然后我重新安装了一下旧版本的VS。

这里要提一下，目前官网旧版本是要订阅的，自己要找一下资源。我的是在B站找到的网盘链接。
重新启动之后：

为了防止因为重复加载编译环境而产生潜在的冲突，脚本要求我们手动设置一个“确认标志”，来告诉它：“是的，我就是要用这个环境，请继续！”。那么我们按它说的做，先设置set DISTUTILS_USE_SDK=1
