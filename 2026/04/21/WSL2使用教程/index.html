WSL2使用教程：在Windows上打造Linux开发环境
更新于 2026-04-21 教程 开发环境

在用 Windows 做开发这件事上，我折腾过不少方案。装双系统嫌麻烦，用虚拟机又觉得太重，Git Bash 凑合能用但总有些命令跑不起来。直到开始用 WSL2，才觉得这个问题算是真正解决了——在 Windows 里开一个终端，就是完整的 Linux 环境，文件可以互通，VS Code 可以直接接进去，几乎感觉不到边界。

这篇文章记录我从安装到配好开发环境的完整过程，适合有一定 Linux 基础、想在 Windows 上用 WSL2 做日常开发的同学。

1. 什么是 WSL2

WSL（Windows Subsystem for Linux）是微软官方提供的在 Windows 上运行 Linux 的方案。WSL2 是第二代，底层基于 Hyper-V 轻量虚拟机，运行的是完整的 Linux 内核，而不是像 WSL1 那样做系统调用翻译。

简单对比一下两代的区别：

特性 WSL1 WSL2
Linux 内核 模拟（系统调用翻译） 完整真实内核
文件系统性能（Linux 文件） 较慢 快
Windows 文件访问性能 快 较慢
网络 与 Windows 共享 独立虚拟网卡（新版 Windows 支持 localhost 自动转发）
兼容性 部分系统调用不支持 接近原生 Linux

对于日常开发来说，WSL2 的兼容性和性能都更好，推荐直接用 WSL2。

2. 安装与基础配置

2.1 前置条件

  • Windows 10 版本 2004（Build 19041）及以上，或 Windows 11
  • BIOS/UEFI 中开启虚拟化（Intel VT-x 或 AMD-V），大多数现代电脑默认已开启
  • 以管理员身份运行 PowerShell 或命令提示符

可以在任务管理器 → 性能 → CPU 中确认”虚拟化”是否显示”已启用”。

3.3 网络与端口

WSL2 和 Windows 共享 localhost，在 WSL 里启动的服务可以直接在 Windows 浏览器里通过 localhost:端口号 访问，反过来也一样。比如在 WSL 里跑一个 Flask 应用监听 5000 端口，Windows 浏览器直接访问 http://localhost:5000 就能看到。

如果遇到访问不通的情况，检查一下 WSL 的防火墙规则，或者确认服务监听的是 0.0.0.0 而不是 127.0.0.1。
